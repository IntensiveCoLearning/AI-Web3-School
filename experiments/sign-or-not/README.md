# 签还是不签 — Web3 合约授权安全训练

## 1. 它解决什么学习问题

Web3 新手最常见的安全事故是看不懂合约授权就直接签名。这个工具通过模拟真实场景，帮用户练习识别危险授权请求，建立「签之前先想清楚」的习惯。

## 2. 用户如何与它交互

CLI 工具，启动后展示一条模拟的合约授权请求（包含合约地址、请求权限、来源网站），用户输入「签」或「不签」，AI 分析风险并告知判断是否正确。

```bash
# 设置 DeepSeek API Key
export DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx

# 启动
cd experiments/sign-or-not
node index.js
```

启动后随机展示案例，不做完 5 个不重复。

## 3. 输入示例和输出示例

**输入：** 某 DeFi 网站请求 approve 无限额度 USDT，用户输入「签」

**输出：** AI 判定高风险，说明无限额度授权意味着对方可随时转走全部资产，用户判断错误。

## 4. 哪部分 AI 生成，哪部分人工验证

- **测试案例场景**：由我（2758star）设计，覆盖安全 / 中风险 / 高风险 / 极高风险四个等级
- **风险分析**：由 DeepSeek API 生成，根据用户的选择给出个性化反馈
- **输出结构和判断逻辑**：由我人工审核修改，确保风险提示准确

## 5. 限制和下一步

- **当前限制**：案例固定内置 5 个，不能输入真实合约地址分析
- **下一步**：接入链上数据，支持用户输入真实授权请求自动分析
