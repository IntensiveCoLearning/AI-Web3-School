## 1. LLM（大语言模型）

**一句话**：一个超级「猜下一个词」的模型，参数够多、训练数据够大之后，在很多场景下看起来像真的理解了语言。

**例子**：你问 DeepSeek「ETH Gas 是什么」，它不是在翻字典，而是基于训练数据里类似问题后面跟着的词，一步步生成回答。

**常见误区**：觉得 LLM「知道」东西。其实它没有任何真正的知识和信念，只是在做概率预测。你问它 1+1 等于几，它不是算出来的，是见过太多次「1+1=2」这个模式。

---

## 2. Prompt（提示词）

**一句话**：你给 AI 的指令，决定了它用什么语气、什么角度、什么格式回答你。

**例子**：问「Gas 是什么」得到教科书回答；改成「你是一个 Web3 新手导师，用买菜比喻解释 Gas，三句话以内」，回答变成了「Gas 就是链上做事的手续费，就像你打车，给的钱越多司机跑得越快」。

**常见误区**：以为 prompt 写得越长越好。有时候太长反而让模型搞不清重点。关键是角色、任务、格式三要素说清楚。

---

## 3. Context Window（上下文窗口）

**一句话**：模型一次能「记住」多少对话内容，超出就失忆了。

**例子**：你跟 Claude Code 连续聊了三个小时，前面说过的内容它还能记住——直到上下文窗口满了，前面的内容就被裁掉了。

**常见误区**：以为上下文越大越好。大了能装更多，但也更慢更贵，而且模型对长文本中间部分的注意力会下降（「中间丢失」问题）。

---

## 4. Agent（智能体）

**一句话**：给 AI 配上工具和使用工具的判断力，让它不只是聊天，还能干活。

**例子**：Hermes Agent 被配置成我的学习代理——它能读 GitHub、调 WCB API、生成打卡草稿、定时提醒。它不是只是回答问题，而是能做一连串操作。

**常见误区**：以为 Agent 就是自动化的万能助手。实际上 Agent 的每一步都是「看到结果 → 判断下一步」，一旦中间某步出错，后续全乱。而且 Agent 不会自己判断「该不该做」，权限控制全靠规则。

---

## 5. Tool Use（工具调用）

**一句话**：AI 在执行过程中调用外部工具（API、数据库、代码执行器）来完成自己做不到的事。

**例子**：交易解释器里，LLM 自己读不了链上数据——是 ethers.js 先把链上数据查出来，整理好，再喂给 LLM 生成解释。LLM 只做最后一步「翻译」。

**常见误区**：混淆「AI 做到的」和「工具做到的」。工具调用是 AI 和外部能力的分界线，用户需要搞清楚哪部分是谁干的。

---

## 6. AI Coding（AI 编程）

**一句话**：用 AI 辅助写代码，从补全一行到生成整个项目。

**例子**：这个交易解释器就是 Claude Code 帮我写的——我描述了想法，它写了全部代码。但我需要确认它没有引入安全问题（比如不小心把私钥写死进代码）。

**常见误区**：生成的代码就能跑。AI 写的代码经常有边界情况没处理、版本不对、依赖路径假定错误。每段 AI 代码都得测试过才能信。

---

## 7. Guardrails（护栏）

**一句话**：限制 AI 不能做什么的规则，比提示词更硬。

**例子**：交易解释器的护栏——只做 `eth_call` 读操作，API Key 从本地文件读取但不输出到公开文档，WCB 提交前必须人工确认。

**常见误区**：以为护栏是系统级的就安全了。护栏规则是代码写的，代码是 AI 写的可能有 bug，最终还得人审核。
