````markdown
## 一、三件套的逻辑关系

```
随机数(256bit)
    │  椭圆曲线点乘（secp256k1）
    ▼
  公钥
    │  Keccak-256 哈希 → 取后 20 字节
    ▼
  地址（你的链上身份）
```

| 概念 | 类比 | 关键性质 |
|------|------|----------|
| 私钥 | 印章原件 | 单向生成公钥，不可逆推 |
| 公钥 | 印章图样 | 可公开，用于验证签名 |
| 地址 | 银行卡号 | 公钥哈希截取，20 字节 |

> **认知纠正**：公钥 ≠ 地址。地址是公钥经过 Keccak-256 再截取后 20 字节的结果。

---

## 二、哈希函数：信息的"指纹机器"

不要把哈希理解为加密——它**不可逆**，更像指纹。

**Keccak-256 的四个关键性质：**

1. **确定性** — 同样输入必得同样输出
2. **雪崩效应** — 改变 1 bit，输出面目全非
3. **不可逆** — 无法从输出反推输入
4. **抗碰撞** — 极难找到两个不同输入产生相同哈希

**在 Web3 里的用途：**
- 把任意长度的交易数据压缩成 32 字节摘要
- 对摘要签名（而非对整笔数据签名），效率高，安全性一致
- 生成地址、构建 Merkle Tree、区块哈希链接

---

## 三、非对称加密：椭圆曲线凭什么安全？

以太坊使用 **secp256k1** 椭圆曲线：

```
私钥 k（随机整数）
公钥 K = k × G     ← G 是曲线的"生成点"，公开常数
```

**为什么单向？**
从 `k` 计算 `K` 是重复做椭圆曲线加法，容易；
从 `K` 反推 `k` 是"椭圆曲线离散对数问题"，目前没有高效算法，
即使用现有最快的计算机穷举，也需要远超宇宙年龄的时间。

> **关键认知**：私钥 = 所有权本身，不是"密码"。没有"忘记密码重置"——丢了就是丢了。

## 四、数字签名：ECDSA 的工作流程

### 签名（发送方）

```
① 构造交易对象（from, to, value, nonce, gasPrice, gasLimit）
② 对交易数据做 Keccak-256 → 得到 32 字节摘要 h
③ 用私钥 k 对 h 做 ECDSA 签名 → 产生 (r, s, v)
④ 把原始交易 + 签名 (r, s, v) 广播到网络
```

### 验证（节点）

```
① 收到原始交易 + (r, s, v)
② 重新计算交易哈希 h
③ ecrecover(h, r, s, v) → 恢复出公钥 → 算出地址
④ 对比 from 字段：地址匹配 → 合法
⑤ 检查 nonce、余额 → 打包进区块
```

**签名三个值的含义：**

| 值 | 含义 |
|----|------|
| `r` | 签名时随机点的 x 坐标 |
| `s` | 签名证明值（与私钥、r、消息哈希相关） |
| `v` | 恢复标识符（27 或 28），用于 ecrecover 确定唯一公钥 |

> **关键认知**：签名不暴露私钥本身。验证者只需公钥，无需任何中间人参与。

## 五、一笔交易的完整字段

```json
{
  "from":     "0x742d35Cc...",   // 发送方地址（从签名恢复，非显式信任）
  "to":       "0xAbCd1234...",   // 接收方地址
  "value":    "1.5 ETH",         // 转账金额
  "nonce":    42,                // 防重放攻击（每笔交易唯一递增）
  "gasPrice": "25 Gwei",         // 每单位 Gas 的出价
  "gasLimit": 21000,             // 最多消耗多少 Gas
  "chainId":  1,                 // 防跨链重放（EIP-155）
  "r": "0x9f71...",
  "s": "0x4a5b...",
  "v": "0x1b"
}
```

**Gas 费计算：**
```
手续费 = gasPrice × 实际消耗 Gas
最大花费 = gasPrice × gasLimit
普通转账固定消耗 = 21,000 Gas
```

## 六、常见认知陷阱

| 误解 | 正确理解 |
|------|----------|
| 公钥 = 地址 | 公钥经 Keccak-256 再取后 20 字节才是地址 |
| 签名 = 加密内容 | 签名不加密，只证明"是私钥持有者发出的" |
| 私钥泄露可以更换 | 私钥与地址绑定，只能弃用该地址 |
| 链上交易内容保密 | 链上数据全部公开，匿名靠地址本身，不靠加密 |
| nonce 可以随便填 | nonce 必须严格递增，否则交易被拒绝或可被重放 |
````
