LLM 本质 → 概率模型，不是事实来源

"模型输出是候选结果，不是事实本身"

关键：越靠近执行层，越要对接外部校验

六大知识节点：

\- Token — 模型的最小处理单位，代码/JSON 比普通段落更吃 token

\- Embedding — 向量化衡量语义接近度，适合检索不适合事实判断

\- Transformer — Attention 模式组合能力强，但不是稳定数据库

\- Hallucination — 在带执行能力的系统里=流程风险，不能只靠 prompt 解决

\- Multimodal — 让模型读懂截图/图表等真实界面，但关键判断仍需可信源

\- 架构位置 — LLM 在理解/生成层，产品还需数据+编排+执行+安全层配合

最小实践方向：

做一个"交易解释器"——核心目标是把模型生成、链上事实、来源边界、不确定性四者分开

模型输出是候选结果，不是事实本身；模型能力是推理入口，不是最终验证。
