workflow 可以理解成“工作流”。

以前很多事情是人一步一步操作，现在会变成：用户输入 → AI 分析 → 调用工具 → 输出结果

这种自动化流程。很多 AI 产品其实本质上都是 workflow 的封装。

对于agent 理解是：如果 prompt 只是“问一次答一次”，那 agent 更像“会自己干活的 AI”。

tool use（工具调用），因为 AI 自己其实不能直接获取实时信息，所以很多时候需要“外挂”。

比如：

-   调用搜索引擎
    
-   调用数据库
    
-   调用计算器
    
-   调用代码执行环境
    

关于web3的概念之前参加过ETHpanda实习计划，有过系统了解。

这些都属于 tool use
