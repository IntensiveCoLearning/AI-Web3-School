# Week 1｜Web3 向进阶：比较 EOA、智能账户、多签账户的权限差异

在 Web3 中，账户不只是"地址"，更重要的是它背后的权限控制方式。EOA、智能账户和多签账户的核心差异，主要体现在：谁能控制账户、谁能发起交易、是否需要多人确认，以及风险由谁承担。

---

## 一、三类账户的基本理解

### 1. EOA：外部拥有账户

EOA 是最常见的钱包账户，例如 MetaMask 中由助记词或私钥控制的普通地址。它的控制权完全来自私钥。

只要掌握了私钥，就可以发起交易、转账、授权合约或签名消息。因此，EOA 的安全边界非常清晰：私钥安全，账户就安全；私钥泄露，账户就基本失控。

### 2. 智能账户

智能账户是由智能合约控制的账户。它不像 EOA 那样只依赖一个私钥，而是可以在合约代码中设置更复杂的权限规则，比如社交恢复、每日限额、自动支付、批量交易、白名单地址等。

智能账户的重点不是"谁拿着一个私钥"，而是"合约规则允许谁做什么"。因此，它的安全边界从单一私钥，扩展到了私钥、合约代码、权限策略和外部服务。

### 3. 多签账户

多签账户通常也是一种智能合约账户，但它的核心特点是：一笔交易不能由一个人单独决定，而是需要多个授权人中的若干人共同确认。例如 3 个管理员中至少 2 个同意，交易才能执行。

多签账户常用于团队金库、DAO 资金管理、项目方资产管理等场景。它的安全边界不在单个私钥，而在签名门槛、成员管理和多人协作流程。

## 二、对比表格

| 比较维度 | EOA | 智能账户 | 多签账户 |
|---------|-----|---------|---------|
| 谁持有控制权 | 单个私钥或助记词持有人 | 由智能合约规则决定，可以是一个或多个密钥、恢复人、自动化模块 | 多个签名人共同控制 |
| 谁可以发起交易 | 私钥持有人可以直接发起 | 符合合约规则的操作者可以发起，可能包括账户本人、授权设备、自动化服务等 | 任一签名人通常可以发起提案，但需要达到确认门槛才能执行 |
| 是否支持多人确认 | 默认不支持 | 可以支持，取决于账户合约设计 | 默认支持，是多签账户的核心功能 |
| 是否支持恢复机制 | 默认不支持。私钥丢失通常意味着资产无法找回 | 可以支持社交恢复、备用密钥、守护人恢复等 | 可以通过更换签名人、调整门槛等方式降低单点丢失风险 |
| 是否支持限额策略 | 默认不支持 | 可以设置每日限额、单笔限额、白名单地址等 | 可以设置，但通常需要通过合约或模块实现 |
| 是否支持自动化 | 默认能力有限，需要用户手动签名 | 可以支持自动扣款、定时交易、批量操作、Gas 代付等 | 可以结合模块实现，但通常更偏重安全审批 |
| 典型使用场景 | 个人钱包、日常转账、DeFi 交互、NFT 交易 | 面向普通用户的智能钱包、游戏账户、订阅支付、账户恢复 | 团队金库、DAO 财库、项目方资金管理、公司链上资产管理 |
| 主要风险点 | 私钥泄露、助记词丢失、钓鱼签名、恶意授权 | 合约漏洞、权限策略配置错误、恢复人被攻击、依赖外部服务 | 签名人串通、签名人失联、门槛设置不合理、治理流程低效 |

## 三、权限差异分析

### 1. EOA 的权限特点

EOA 的权限结构最简单：谁拥有私钥，谁就拥有账户控制权。

这带来的好处是使用简单、成本低、兼容性强。大多数钱包、DApp 和链上应用都原生支持 EOA。用户只需要保管好助记词或私钥，就可以完成转账、授权、签名等操作。

但问题也很明显。EOA 几乎没有内置保护机制。一旦用户在钓鱼网站签署恶意授权，或者私钥被木马、假钱包、云端泄露，攻击者就可以直接转走资产。EOA 也不支持原生恢复，如果助记词丢失，账户中的资产通常无法找回。

**适用场景**：个人日常使用，例如小额转账、连接 DeFi、购买 NFT、测试链交互等。

**主要风险点**：私钥或助记词一旦泄露，攻击者可以直接控制账户；用户误签恶意交易也可能造成资产损失。

**必须人工确认的操作**：转账、合约授权、签名消息、与 DApp 交互等关键操作通常都需要用户用钱包手动确认。

### 2. 智能账户的权限特点

智能账户的权限由智能合约控制，因此它可以实现比 EOA 更灵活的账户管理方式。

例如，一个智能账户可以规定：小额支付可以快速通过，大额转账必须二次确认；某些地址可以加入白名单；如果主设备丢失，可以由几个可信联系人帮助恢复账户；也可以把多个操作打包成一笔交易，减少用户操作复杂度。

智能账户的优势在于它可以把传统互联网账户中的一些体验带入 Web3，比如找回账户、权限分级、自动支付和风险控制。它更适合普通用户大规模使用，因为用户不一定需要理解所有底层签名细节。

不过，智能账户也引入了新的风险。因为权限逻辑写在合约里，所以如果合约代码有漏洞，或者权限模块配置错误，攻击者可能利用规则缺陷转移资产。此外，社交恢复、自动化服务、Paymaster 等外部模块也可能成为新的攻击面。

**适用场景**：智能钱包、链游账户、订阅支付、需要账户恢复的新手用户钱包、需要复杂权限控制的应用账户。

**主要风险点**：智能合约漏洞、恢复机制被滥用、权限策略设置错误、自动化模块或第三方服务失效。

**必须人工确认的操作**：首次设置账户规则、添加或移除恢复人、提高支付限额、大额转账、授权高风险合约等操作应当人工确认。部分低风险、小额或白名单内操作可以根据策略自动执行。

### 3. 多签账户的权限特点

多签账户强调的是"多人共同批准"。它解决的是单人控制资金带来的风险。

例如，一个团队金库可以设置为 3/5 多签，即 5 个成员中至少 3 个确认，资金才能转出。这样即使其中一个人的私钥泄露，攻击者也无法单独转走资产。

多签账户非常适合组织使用，因为组织资金通常不应该由一个人完全控制。多签机制可以增加透明度，也可以让团队内部形成更明确的审批流程。

但多签并不等于绝对安全。如果签名人之间串通，仍然可以转走资金。如果签名门槛太高，可能因为成员失联导致账户无法操作；如果门槛太低，又会降低安全性。多签账户还依赖成员对交易内容的理解，如果签名人盲目确认，也可能批准恶意交易。

**适用场景**：DAO 财库、项目方资金账户、公司链上资产管理、团队共同管理的 NFT 或 DeFi 资金池。

**主要风险点**：签名人串通、签名人私钥被盗、签名人失联、门槛设置不合理、审批效率较低。

**必须人工确认的操作**：资金转出、修改签名人、调整确认门槛、升级合约、授权重要合约等操作必须由达到门槛数量的签名人手动确认。

## 四、哪些操作必须人工确认？

不同账户类型对人工确认的要求不同，但一般来说，以下操作都应该由人进行明确确认：

| 操作类型 | 是否应人工确认 | 原因 |
|---------|:---:|------|
| 转出资产 | 是 | 直接影响资产所有权 |
| 授权合约花费 Token | 是 | 恶意授权可能导致资产被转走 |
| 签署不清楚内容的消息 | 是 | 可能被用于登录、授权或链下攻击 |
| 修改账户控制人 | 是 | 会改变账户控制权 |
| 修改多签门槛 | 是 | 会改变安全模型 |
| 添加或删除恢复人 | 是 | 影响账户恢复权限 |
| 提高自动支付限额 | 是 | 可能扩大损失范围 |
| 小额白名单支付 | 不一定 | 如果提前设置过安全策略，可以自动执行 |
| 定时任务或自动化交易 | 不一定 | 前提是用户已事先确认规则和范围 |

---

## 五、总结

EOA、智能账户和多签账户代表了三种不同的权限模型。

EOA 最简单，适合个人使用，但风险集中在单个私钥上。智能账户更灵活，可以支持恢复、限额、自动化和复杂权限策略，适合提升普通用户体验。多签账户更强调多人共同治理，适合团队、DAO 和机构管理大额资产。

**从安全边界来看：**

- **EOA 的安全边界是私钥。** 只要私钥安全，账户就安全；私钥泄露，账户基本失控。
- **智能账户的安全边界是合约规则。** 它可以降低单点私钥风险，但需要保证合约代码和权限配置安全。
- **多签账户的安全边界是签名人和确认门槛。** 它降低了单人作恶或单点泄露的风险，但也需要合理设计签名人数、门槛和审批流程。

因此，选择哪种账户并不是单纯看哪种"更高级"，而是要看使用场景。如果是个人小额日常操作，EOA 足够方便；如果需要恢复、自动化和更好的用户体验，智能账户更合适；如果是团队或组织管理资金，多签账户通常更安全。
